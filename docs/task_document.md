# AutoVisionTest - 开发任务文档 v1.0

---

## 1. 项目概述

### 1.1 项目背景

配合 AI 编程工作流，开发者使用 AI 编码工具（如 Cursor、Copilot）修改代码后，需要自动化验证修改是否正确。当前缺少一套能在本地运行的、纯黑盒的桌面应用测试方案。

### 1.2 项目目标

开发一套完整的 AI 视觉驱动桌面应用自动化测试框架，具备以下能力：
- 支持纯黑盒测试，无需修改被测应用
- AI 视觉模型本地部署，无需云端 API
- 与 AI 编程 Agent 无缝集成，形成闭环
- 支持确定性测试和探索性测试两种模式

### 1.3 约束条件

- **纯黑盒**：不依赖被测应用的 Accessibility API、Instrumentation 或任何注入方式
- **本地部署**：VLM 模型必须运行在本地 GPU 上，不调用云端服务
- **硬件门槛**：最低 8GB 显存（RTX 3060 或同等性能显卡）
- **隐私安全**：所有测试数据（截图、日志）仅保存在本地

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 目标 | 说明 |
|------|------|------|
| 单步操作延迟 | < 3 秒 | 从截图到操作执行完成 |
| 模型首次加载时间 | < 30 秒 | 模型冷启动时间 |
| 内存占用 | < 8GB | 不含模型显存，VLM 显存另计 |
| 磁盘占用 | < 500MB | 框架本体，不含模型 |

### 3.2 可靠性需求

| 指标 | 目标 | 说明 |
|------|------|------|
| 视觉识别准确率 | > 90% | 常见 UI 元素识别 |
| 误操作率 | < 5% | 错误点击 / 输入比例 |
| 断言准确率 | > 95% | 判断测试通过 / 失败的准确度 |
| 死循环防护 | 100% | 必须有可靠的终止条件 |

### 3.3 兼容性需求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11（优先）、macOS（待支持）、Linux（待支持） |
| 被测应用 | 所有基于 GUI 的桌面应用，不限技术栈 |
| Python 版本 | 3.10, 3.11, 3.12 |
| GPU | NVIDIA CUDA 12.x + cuDNN 8.x |

## 5. 技术选型细节

### 5.1 VLM 模型对比

| 模型 | 参数量 | 显存需求 | 优势 | 劣势 |
|------|--------|---------|------|------|
| ShowUI-2B | 2B | 8GB | 专为 GUI 设计、轻量 | 中文支持弱、社区小 |
| Qwen2-VL-2B | 2B | 8GB | 阿里开源、中文强、社区活跃 | GUI 专用性不如 ShowUI |
| Qwen2-VL-7B | 7B | 16GB | 精度更高 | 显存要求高、速度慢 |

**推荐**：首期使用 Qwen2-VL-2B-Instruct，兼顾性能和兼容性。二期可增加 ShowUI-2B 作为备选。

### 5.2 推理框架对比

| 框架 | 优点 | 缺点 |
|------|------|------|
| vLLM | PagedAttention 高效吞吐、持续批处理 | 资源占用较大 |
| llama.cpp | 纯 CPU/GPU、启动快、资源占用小 | VLM 支持不如 vLLM 成熟 |
| transformers | 灵活、调试方便 | 吞吐量低 |

**推荐**：首期使用 vLLM，二期增加 llama.cpp 支持。

## 6. 验收标准

### 6.1 功能验收

| 功能 | 验收条件 |
|------|---------|
| 确定性用例执行 | 给定 YAML 用例，能正确执行每步操作并返回 PASS/FAIL |
| 探索性用例执行 | 给定自然语言目标，AI 能自主规划并完成测试 |
| 视觉感知 | 能正确识别常见 UI 元素（按钮、输入框、菜单等） |
| 桌面控制 | 能正确模拟鼠标、键盘操作 |
| 测试报告 | 能生成符合 JSON 协议格式的测试报告 |
| API 接口 | FastAPI 能正确接收请求并返回测试结果 |

### 6.2 性能验收

| 指标 | 验收条件 |
|------|---------|
| 单步延迟 | VLM 模型常驻后，单步操作 < 3 秒 |
| 稳定性 | 连续执行 100 次相同用例，无崩溃、无死循环 |
| 准确率 | 10 个标准用例，视觉识别准确率 > 90% |

### 6.3 集成验收

| 集成项 | 验收条件 |
|--------|---------|
| AI 编程 Agent | 测试结果 JSON 能被 AI 编程 Agent 解析并生成修复建议 |
| CI/CD | 能接入 GitHub Actions，接收 pull request 触发测试 |
| 本地模型 | 模型完全本地运行，无任何云端 API 调用 |

## 7. 风险与缓解

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| VLM 模型识别精度不足 | 高 | 中 | 多级感知兜底（像素 → OCR → VLM），UIA 可选增强 |
| 测试执行超时 / 死循环 | 高 | 中 | 严格的七层终止条件兜底 |
| 不同桌面应用 UI 差异大 | 中 | 高 | 分层抽象桌面控制层，适配器模式支持多平台 |
| GPU 资源不足 | 高 | 中 | 提供 CPU fallback 模式（精度换速度），明确最低硬件要求 |
| AI 推理延迟影响实时性 | 中 | 低 | 模型常驻内存 + 异步推理 + 截图压缩优化 |
